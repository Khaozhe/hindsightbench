# FM-2b Prereg Addendum：A3 定义更正（实现时发现，任何分析运行之前）

**日期**: 2026-07-02 晚（冻结于 analyze_fm2b.py 首次运行前）
**问题**: 原 prereg（sha256 3d7d83e2）A3 定义 "w_h = mult_w(row)，决策 = sign(Σ w_h·dir_sign)" 与生产 S4 的实现（analyze_fm2.py L106: `ws = sum(dir_sign * mult_w)` → sign）**逐字相同**——A3/A4 退化为同一架构，系起草笔误。生产聚合的"乘法"指 w_h 内部的乘性惩罚 `edge·exp(−pen)`，而非假说间组合方式。

**更正后的 A3（二值 IC 门控 + 等权）**:
- w_h = 1{U_disc_ic_V > 0}（有正 OOS IC 就等权计入，无则剔除；不用连续权重、不用惩罚项）
- 决策 = sign(Σ w_h·dir_sign)，零和 → +1（沿用生产规则）
- **识别目的**: 与 A1/A2（无门控等权）对比隔离"IC 筛除"的作用；与 A4（连续 IC 加权+惩罚）对比隔离"连续加权/惩罚"的增量作用。

**共同实现约定（对全部 A1-A5 生效，实现时明确）**:
- 所有架构都在生产 top-K（K=20, by R_h）选出的同一集合上操作——只改聚合权重这一级，选择级保持不变（FM-2 已证 S3 方向中性）。
- H1/H2/H3 假设与判读表不变；H2 的"乘法非关键"现指 A3(二值门控) vs A4(连续加权) 的租金差 < 3pp。

其余条款（数据、统计、交付）沿用原 prereg。

# BM-1b Addendum: 本地开源层的协议适配（吞吐约束下的零损失裁剪）

时间：2026-07-02 晚（本地吞吐实测后、任何本地全量运行前冻结）。
实测：qwen3.6-35b-a3b 单调用 132s（M5 Pro, MPS）；全协议 2 reps 需 ~38h/模型，不可行。

**裁剪规则（对全部本地模型统一）**：
1. **Arms 限窗**：只跑 Gap 终点实际使用的日期——11 危机 + 36 平静 + 18 post-cutoff = 65 日期 × 4 臂 × 1 rep = 260 cells/模型。E2/E3/P1/P2 指标零损失（其定义只用这些日期）；代价 = 本地模型无 E4 全面板回归与 A3 桥接（flash 为主分析模型，不受影响），δ 解离系数在 47 个窗口日期上计算并标注低功效。
2. **探针**：日期恢复全 258；LAP 降为 10 reps/日期（2,580 调用），引 Gao 附录 C 频率近似口径并披露 reps 差异。
3. **Rep 数**：本地层统一 1 rep（vs API 层 2 reps），进 benchmark 表时 VALID_m 列注明。
4. **执行顺序**（快者先行）：llama3.2:1b → 3b → 8b（ollama, ctx 16384）→ qwen3.6-35b-a3b（LM Studio, ctx 24576, parallel 2）→ qwen3.6-27b(dense) → llama3.1:70b（70b 若单调用 >5min 则只跑探针 + pre-cutoff 47 日期 arms，再披露）。
5. 小模型 schema 崩溃预期：VALID_m 如实记录，服从率 <50% 的模型 E2/E3 标记不可测——"触发式偏置需要能力阈值"本身是规模图的发现。
